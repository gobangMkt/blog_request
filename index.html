# FAQ 작성 가이드 (FAQ 모음 DB)
📌 이 문서는 **새 FAQ 문서(페이지)**를 작성할 때, 기존 문서 톤/구조와 동일하게 맞추기 위한 작성 지침입니다.
## 1) 기본 원칙 (한 문서 = 한 질문) - **페이지 제목(Name)**은 사용자가 실제로 물을 법한 **질문 문장 1개**로 작성합니다. - 예: “광고는 언제부터 시작되나요?”ㄱ,랫 - 예: “세금계산서 발행이 가능한가요?” - 답변 본문은 **즉답 → 조건/예외 → 다음 액션(요청 방법) → 참고** 순서로 짧게 구성합니다. - 문장 길이는 짧게, 핵심 문구는 **굵게** 처리합니다. ## 2) DB 속성 입력 규칙 - **Tags**: 해당 질문의 성격을 1개 중심으로 선택합니다. - 수정 / 광고 / 결제 / 등록 / 기타 - **Created**: 자동 생성(수정 불필요) ## 3) 권장 답변 템플릿 (복붙용) ``` (질문에 대한 두괄식 답변 작성) (선택) **(사용자 상황에 대한 짧은 공감/분기 질문)**
(분기/예외/대안 안내) (요청/신청 방법: 어디로, 무엇을 남기면 되는지) (선택) 💡** 참고하세요!** (오해가 많은 포인트 / 정책 근거 / 대체 증빙 등) ``` ## 4) 문체/형식 스타일 가이드 ### 4-1. 강조/가독성 - 핵심 결론은 첫 줄에 두고, 반드시 **굵게**로 강조합니다. - 줄바꿈이 필요하면 일반 줄바꿈 대신 **``**를 사용해 한 블록 안에서 정리합니다. - 안내 문장 중 중요한 정책/조건은 색상 강조를 제한적으로 사용합니다. ### 4-2. 구조(권장 순서) 1. **즉답** 2. **상황별 분기/예외** (필요할 때만) 3. **사용자가 해야 할 행동** (담당자/채널톡/대표번호/신청 링크 등) 4. **참고(법적 효력/증빙 대체 등)** (필요할 때만) ### 4-3. 단어/표현(일관성) - “가능/불가능”을 애매하게 쓰지 말고 **명확하게** 씁니다. - 예: “발행이 **불가능**합니다” - 정책/조건 문장은 단정형으로 씁니다. - 예: “결제 완료 시점부터 즉시 시작되어 노출됩니다.” ## 5) 체크리스트 (발행 전 30초 점검) - [ ] 제목이 “질문형”이고 너무 길지 않다. - [ ] 첫 줄에 결론(즉답)이 있다. - [ ] 사용자가 다음에 무엇을 하면 되는지(요청 경로)가 명확하다. - [ ] Tags가 1개 중심으로 들어갔다. - [ ] 불필요한 내부 용어/메모가 없다. ## 6) 예시 (현재 DB 톤 참고) - 광고 시작일/예약 안내 스타일: “광고는 언제부터 시작되나요?”[[1]](https://www.notion.so/3180d7602cd481e29783c419a59bfd75?pvs=21) - 결제/증빙 안내 스타일: “세금계산서 발행이 가능한가요?”[[2]](https://www.notion.so/3180d7602cd481cd9f46d66998bf1f6e?pvs=21) - 정책/조건 강한 공지 스타일(색상/구분선): “고방에 지점 노출이 안돼요.”[[3]](https://www.notion.so/3180d7602cd48139bdb7f1601d22a51f?pvs=21)
